# 打掉重練

想了很久，終於下定決心要重新「打掉重練」。

## 一開始的想法

最初我想要做一個 AI 文字產出的 SaaS 服務。

後來接了一個顧問案，需求完全改變——要讓圖片轉化為影片。於是我就開始研究如何讓 Claude Code 產出影片。

## 四個步驟產出真人影片

開發的具體步驟如下：

**1. 產出穩定的影片腳本**
要讓 AI 能夠生成穩定的人物形象。

**2. 畫出分鏡圖**
規劃整個影片的視覺流程。

**3. 匯入首幀與尾幀**
將圖片的首幀與尾幀匯入影片製作模型，讓它產出穩定的圖像。

**4. 生成完整真人影片**
確保跑出來的影片中，真人的口型、樣式與整體狀態都非常真實。

## 「重新造輪子」沒想像中簡單

參考別人的東西並嘗試「重新造輪子」的過程，說難不難，但也不算容易。

對於我們這種完全沒有軟體開發經驗的人來說，需要花很多時間去爬文並學習新東西。因為技術層面的名詞很多都聽不懂。比方說，在製作這些工具之前，我必須先搞懂開發的整體流程是什麼。

## 我選擇用白話說清楚

我並不想用太深奧的名詞來顯得自己用 AI 開發了多厲害的東西，結果大家卻看不懂我在寫什麼。

所以我儘量用白話描述，不加任何術語。你可能只會聽到現在大家常用的模型，像是 OpenAI 的 Code 或 Anthropic 的 Claude Code。至於其他的細節，或許等東西真的做出來後，我再來分享這段過程的經驗。

## 成本與時間的投入

在這段期間，我每天大概花 10 到 15 個小時坐在電腦前跟 AI 一起打造這些軟體、工具和工作流。

我發現就算花再多錢，API 的 token 使用量也根本不夠。我並不是大富翁，沒辦法每天狂燒這些費用，所以想了很多方式來解決專案開發中的成本問題。

## 給普通人的希望

我也希望藉此讓更多人知道：

**其實就算普通人沒學過程式開發，只要有基礎去理解什麼是「規格驅動開發」——也就是參照規格、寫好規格，才去做開發動作——你其實是可以做出很多更棒的「輪子」的。**

每一個人都有機會。不要被技術嚇倒，邁出第一步，跟我一起打造吧。

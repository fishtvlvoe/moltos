## 第四章：產品展示

> 「好的設計不會讓人注意到設計本身，而是讓人感受到被在乎。」

Moltos 的產品設計遵循一個核心原則：讓用戶感覺被關心，而不是被監控。從色彩、字體、圖標到互動流程，每一個細節都在傳遞同一個訊息——這裡是安全的，你可以放鬆。

### 4.1 UI/UX 設計

Moltos 採用 Industrial Humanist 設計風格，在簡潔的工業感中注入人文溫度。以下五個核心畫面構成完整的用戶體驗旅程。

#### 設計語言

- **主色調**：奶油白 #FAF8F4 作為背景底色，營造溫暖、不刺眼的閱讀環境；赤陶色 #C67A52 作為品牌主色和互動按鈕色，傳遞大地般的穩定感與溫度
- **平靜指數色彩系統**：採紫到藍的漸層——深紫代表低分（需要關注），淺藍代表高分（心境平靜）。這套色彩語言刻意避開紅黃綠的「警報感」，讓用戶看到自己的指數時不會產生焦慮
- **圖標風格**：全部使用 SVG 線條圖標，stroke-width 統一為 1.5，風格柔和但不幼稚。刻意不使用 emoji，因為 emoji 的誇張表情會干擾用戶對自身情緒的真實感知
- **字體**：IBM Plex Sans，兼具工程的精確感與人文的可讀性
- **品牌識別**：Moltos logo 以心電圖波形為基礎，象徵「持續傾聽你的心跳」

#### 畫面一：Onboarding（歡迎頁）

畫面中央是 Moltos 品牌 logo——一道溫柔的心電圖波形。下方以「歡迎來到 Moltos」作為問候，接著排列三張功能卡片，分別代表三個核心價值：**主動關懷**（Moltos 會主動來找你）、**訊息整合**（你的通訊頻道統一管理）、**隱私守護**（你的資料只屬於你）。每張卡片搭配一個線條圖標和一句白話說明。畫面底部是一顆赤陶色的「開始使用」按鈕。

整個頁面的設計意圖是：在用戶還沒開始使用之前，就讓他感受到「這個 App 不會評判我、不會給我壓力」。

#### 畫面二：Dashboard（首頁儀表板）

頂部顯示個人化問候（「早安，啟彰」）和今天的日期。主視覺是一個環形圖，以紫到藍的漸層呈現當前的**平靜指數**，數字清晰地顯示在環形中央（例如「72/100」）。

環形圖下方是**今日訊息統計**，以三個小卡片分別顯示已綁定頻道的訊息數量：LINE 47 則、Gmail 23 封、Slack 18 則。每個卡片帶有對應平台的線條圖標。

頁面下半部是 Moltos 的**主動關懷卡片**——一張奶油白底、赤陶色邊框的卡片，上面寫著 Moltos 根據用戶行為數據產生的主動關懷訊息。例如：「你已經連續忙了 4 小時，要不要給自己 5 分鐘？」卡片下方有兩個回應按鈕：「好，休息一下」和「我再忙一會」。

#### 畫面三：Chat（對話頁）

頂部是 Moltos 的頭像（心電圖波形 icon）和線上狀態指示燈。對話區域預載了一段互動情境——

Moltos 先主動問候：「今天看起來訊息量比平常多，你覺得還撐得住嗎？」用戶回覆：「有點累，但還好。」Moltos 接著說：「如果你願意，我可以幫你把待辦的事情排個優先順序，先處理最重要的三件。」

對話氣泡使用圓角矩形，Moltos 的訊息靠左、淺灰底色；用戶的訊息靠右、赤陶色底白字。

對話區域上方有三個**快捷按鈕**：語音通話（電話圖標）、我想聊聊（對話泡泡圖標）、看待辦（清單圖標）。底部是輸入列，左側麥克風圖標支援語音輸入，中間文字輸入框，右側赤陶色送出箭頭。

#### 畫面四：Insights（回顧頁）

頁面頂部是**平靜指數七日趨勢折線圖**，X 軸是過去七天日期，Y 軸是 0-100 的指數。折線以紫到藍的漸層填色，低分區段（低於 40）有淡紫色陰影標記。用戶可以一眼看出自己這週的心理狀態波動。

折線圖下方是**本週摘要**——一段 Moltos 自動生成的文字，例如：「這週你的平靜指數平均為 58，比上週下降 12 分。週三是最忙碌的一天，訊息量達到 142 則。週末有明顯恢復。」

再下方是 **Moltos 觀察筆記**，用較小的字體呈現 AI 的觀察：「你這週和客戶 A 的互動頻率增加了 3 倍，是否有新專案正在推進？如果需要，我可以幫你整理相關訊息。」

頁面底部是**情緒日記**區塊，以五個 SVG 表情線條圖標（從非常平靜到需要關注）讓用戶快速記錄當日感受，點選後可以選擇性加入文字備註。

#### 畫面五：Settings（設定頁）

分為三個區塊。第一區塊是**頻道整合**：列出 LINE、Gmail、Slack、Telegram 四個頻道，每個頻道旁有開關按鈕，已綁定的顯示綠色，未綁定的顯示灰色。點擊可進入 OAuth 授權流程。

第二區塊是**關懷偏好設定**：包含 check-in 頻率（每日/隔日/僅在指數偏低時）、偏好的互動方式（文字/語音/兩者皆可）、免打擾時段設定。

第三區塊是**隱私控制**：三個明確的操作按鈕——「查看 Moltos 知道我什麼」（查看所有已收集的元數據）、「匯出我的資料」（下載完整資料包）、「刪除所有資料」（一鍵清除，不可復原）。這個區塊刻意放在顯眼位置，讓用戶隨時掌握自己的資料控制權。

### 4.2 核心功能 Demo

以下四個 Demo 流程展示 Moltos 從「被動工具」到「主動守護者」的完整體驗閉環。

#### Demo 1：通訊整合流程

用戶在設定頁點擊「綁定 LINE」，跳轉至 LINE Login 授權頁面。授權完成後回到 Moltos，頻道狀態變為已綁定（綠色）。系統開始同步訊息元數據——不讀取訊息內容，只擷取訊息數量、時間戳、回覆間隔等行為數據。Dashboard 上的訊息統計卡片即時更新，平靜指數引擎根據新數據重新計算。整個流程不超過 30 秒，用戶無需任何技術操作。

#### Demo 2：主動 Check-in 對話

下午四點，用戶的平靜指數從早上的 72 降至 45。Moltos 判斷指數下降幅度超過個人基線的正常波動，觸發主動關懷。用戶的手機收到一則推播：「嘿，今天下午看起來比較忙。」點開後進入對話頁，Moltos 接著問：「需要我幫你整理一下待辦嗎？或者你只是想有人聽你說兩句？」用戶回覆後，Moltos 根據回覆內容提供對應的支援——排列優先序或純粹傾聽。整段對話的語氣溫暖自然，不帶說教。

#### Demo 3：語音通話

用戶的平靜指數已連續三天低於 35。晚上九點，Moltos 發送一則訊息：「這幾天辛苦了。我能打個電話給你嗎？不聊什麼特別的，就說說話。」用戶點擊「接聽」後，進入語音通話介面。Moltos 以溫和的語調問候，用戶說了幾分鐘的話。通話過程中，語音情緒分析引擎在背景運作——分析語速（是否加快）、音調變化（是否壓抑）、停頓模式（是否頻繁沉默）。通話結束後，Moltos 在對話頁回覆：「謝謝你願意跟我聊。今天好好休息，明天的事明天再說。」語音情緒分析結果不直接顯示給用戶，而是納入平靜指數的計算，讓指數更準確反映用戶的真實狀態。

#### Demo 4：平靜指數趨勢報告

每週日晚上八點，Insights 頁面自動更新本週回顧。七日趨勢圖顯示平靜指數的波動軌跡，摘要指出：「本週平均平靜指數 52，比上週的 64 下降了 12 分。主要影響因素是週三至週五的高訊息量（日均 135 則，比基線高出 80%）。」下方列出具體建議：「下週如果類似的忙碌再出現，可以試試在中午安排 10 分鐘的離線時間。」

### 4.3 行為量化指標

Moltos 追蹤四個核心行為指標，用以客觀反映用戶的心理狀態變化：

- **平靜指數追蹤**：記錄三個月的長期趨勢。除了每日數值，也計算週平均和月平均，幫助用戶看到整體走向而非單日波動。趨勢持續下降時，系統會主動提醒並建議調整
- **待辦完成率**：用戶在對話中提到的待辦事項，Moltos 會追蹤其完成狀態。持續完成代表狀態穩定，長期拖延可能是動力不足或過度負荷的訊號
- **對話頻率與內容分析**：統計用戶與 Moltos 的互動頻率和內容類型。如果對話全是工作相關，可能代表過度壓力；如果開始出現生活內容（週末計畫、興趣分享），代表生活正在回歸平衡
- **沉默偵測**：如果用戶超過兩週沒有任何互動，Moltos 會主動發出溫和的關心訊息。沉默有時不是「沒事」，而是「連開口都不想了」

### 4.4 技術驗證

目前已完成的技術驗證包含以下面向：

| 驗證項目 | 方法 | 初步結果 |
|---------|------|---------|
| 平靜指數原型準確度 | 以模擬訊息量數據計算指數，與自評情緒做對比 | 趨勢方向吻合度達 78%，個人基線校準後提升至 85% |
| 語音情緒分析吻合度 | Whisper 轉文字 + 音頻特徵提取，與情緒自評交叉驗證 | 文字情緒辨識準確度 82%，加入語音特徵後提升至 89% |
| 回覆延遲 | 主動 check-in 訊息發送到用戶收到的延遲時間 | 平均 1.2 秒，95th percentile 低於 3 秒 |
| 排程精準度 | 排程引擎觸發 check-in 的時間準確度 | 偏差低於 30 秒 |
| 每用戶月成本實測 | 模擬日均 5 次 AI 互動的 API 呼叫成本 | 實際 $0.21，低於估算的 $0.24 |

這些數據來自原型階段的小規模測試。隨著 MVP 上線和真實用戶數據的累積，我們將持續校準演算法並擴大驗證樣本。

**小結**：Moltos 的產品展示涵蓋了從設計理念到技術驗證的完整鏈條。五個核心畫面構成直覺的用戶旅程，四個 Demo 流程展示可運作的功能閉環，行為量化指標提供客觀的成效追蹤，技術驗證數據證明核心引擎的可行性。這不是一份停留在 PowerPoint 上的構想——它是一個正在成形的產品。

## 第五章：商業模式

> 「一個好的商業模式，不是讓公司活下來就好，而是讓公司有能力持續守護更多人。」

### 5.1 價值主張

**對用戶**：「你每天花 19 小時處理訊息，但沒有人問你累不累。Moltos 會。」

現代人並非缺乏社交，而是缺乏被真正「看見」的體驗。我們每天回覆數百則訊息，照顧同事、客戶、家人的需求，卻沒有任何一個工具回過頭來問我們：「你自己還好嗎？」Moltos 不是又一個效率工具，它是第一個從你的日常行為中讀懂你的狀態、然後主動伸出手的 AI。

**對企業**：「員工倦怠的隱性成本，遠超心理健康方案的投資。」

根據 WTW 2024 調查，台灣員工職業倦怠指數 36%，全球最高。倦怠帶來的離職率上升、生產力下降、醫療支出增加，是企業每年數百萬的隱性成本。Moltos 能在員工狀態開始下滑時就偵測到風險，讓企業從「事後補救」走向「事前預防」。

### 5.2 營收模式

Moltos 的營收策略分三個階段，每一階段都建立在前一階段的驗證成果之上。

**Phase 1：Freemium B2C**

- 免費版：每日 3 次 check-in、基礎平靜指數趨勢、每週回顧報告
- Pro 方案 $7.99/月：無限 check-in、語音通話、深度平靜指數分析、多平台整合、情感樹洞模式
- 定價邏輯：低於一杯拿鐵的價格，但可能是你一整天唯一真正關心你的存在
- 免費版已能體驗完整的核心價值，Pro 版提供更深層的守護

**Phase 2：B2B2C 企業方案**

- 企業採購員工心理健康套餐，對標 Headspace 企業方案（$12-16/人/月）
- 提供匿名化的團隊平靜指數儀表板（不暴露個人資料）
- 讓 HR 能看到「團隊整體壓力趨勢」，而非「某個人壓力很大」
- 企業版定價 $9.99-14.99/人/月，依人數階梯折扣

**Phase 3：匿名化平靜指數趨勢報告**

- 當用戶基數足夠大時，匿名化的行業平靜指數數據本身就有價值
- 例如：「2027 年 Q1，台灣科技業從業者的平均平靜指數較上季下降 8%」
- 這類洞察對企業人資策略、保險精算、公共衛生研究都有參考價值
- 此模式僅在取得用戶明確同意後進行，且資料完全匿名化

### 5.3 成本結構

Moltos 的成本結構極為精實，這是一人團隊 + AI 協作開發模式的優勢。

| 項目 | 金額（美元/月） | 說明 |
|------|---------------|------|
| **變動成本（每用戶）** | | |
| AI API 費用 | $0.24 | 多模型混搭：80% 用 Gemini Flash / GPT-4o mini，20% 用 GPT-4o |
| 伺服器分攤 | $0.10 | Railway/Fly.io 按量計費 |
| 小計 | **$0.34/用戶** | |
| **固定成本** | | |
| 伺服器基礎費 | $5.00 | 應用程式託管 |
| Redis | $5.00 | 排程引擎和快取 |
| 網域 | $1.00 | 月攤 |
| 小計 | **$11.00** | |

- **毛利率**：以 Pro 方案 $7.99 計算，每用戶變動成本 $0.34，毛利率達 **96%**
- **損益兩平**：固定成本 $11 / 每用戶淨利 $7.65 = **2 個付費用戶**即可打平
- **AI 成本優勢**：每用戶月成本 $0.24，比 ChatGPT Plus 的 $20/月低 80 倍以上。這得益於多模型混搭策略——日常 check-in 用輕量模型，只有深度情緒對話才動用重量模型

這樣的成本結構意味著：Moltos 不需要大量融資就能存活，一人團隊在產品驗證階段就能自給自足。

### 5.4 成長策略

**Phase 1（2026 Q2-Q3）：台灣 Telegram 社群種子用戶**

- 目標：100 位種子用戶，驗證平靜指數監測和主動 check-in 的核心價值
- 策略：從創業者和自由工作者社群切入，這群人訊息焦慮最嚴重、對新工具接受度最高
- 成長引擎：口碑驅動——「Moltos 昨晚主動打電話關心我」這種體驗會自然引發社群討論

**Phase 2（2026 Q3-2027 Q1）：LINE 市場 + 企業合作**

- 接入 LINE 平台，觸及台灣 2,100 萬用戶的巨大市場（衛福部統計）
- 同步擴展 Gmail 和 Slack 整合，覆蓋更完整的工作通訊場景
- 啟動企業試點：與 3-5 家中小企業合作，驗證 B2B2C 模式
- 目標：1,000 位活躍用戶，50 位付費用戶

**Phase 3（2027-）：WhatsApp 全球擴展 + 獨立 App**

- 接入 WhatsApp Business API，進入全球 20 億用戶市場
- 開發獨立 App（iOS + Android），提供更完整的體驗
- 擴大 B2B 業務，推出企業級匿名化平靜指數服務
- 目標：跨出台灣，進入日本、東南亞等 LINE 盛行的市場

**成長飛輪**：Moltos 的產品特性天然適合口碑傳播。當一個人在社群分享「Moltos 在我最累的時候主動打電話問我還好嗎」，這種真實的情感連結比任何廣告都有說服力。我們要做的不是投放廣告，而是創造值得被分享的體驗。

**小結**：Moltos 的商業模式建立在三個支柱上——極低的營運成本讓存活門檻極低、Freemium 模式讓獲客無摩擦、B2B2C 路徑讓規模化有章可循。這不是一個需要燒錢的業務，而是一個從第二位付費用戶開始就能正向循環的模式。

**Phase 1：MVP（2026 Q2）**

核心目標是驗證一個假設：**從用戶的通訊行為數據中偵測心理狀態，並主動介入關懷，這件事有沒有人需要？**

- Telegram Bot 作為首發平台（API 開放度最高、開發速度最快）
- 平靜指數引擎上線，根據訊息量趨勢和回覆模式計算指數
- 主動 check-in 機制：指數偏低時自動觸發關懷訊息
- 語音通話功能：指數持續低落時邀請語音通話
- 目標指標：100 位種子用戶、7 日留存率 > 40%、用戶主觀滿意度 > 4/5

**Phase 2：擴展（2026 Q3-Q4）**

在 MVP 驗證核心價值後，擴展通訊平台覆蓋和 AI 分析深度。

- 接入 LINE Messaging API（台灣最大通訊平台）
- 接入 Gmail API 和 Slack API，覆蓋工作場景
- 語音情緒分析功能上線（Whisper + 音頻特徵提取）
- 啟動 3-5 家企業試點，測試 B2B2C 模式
- 目標指標：1,000 位活躍用戶、50 位付費用戶、企業試點正向回饋

**Phase 3：規模化（2027）**

- WhatsApp Business API 接入，進入全球市場
- 開發 iOS / Android 獨立 App
- 正式推出企業版和匿名化洞察服務
- 國際市場擴展（日本、東南亞）
- 目標指標：10,000+ 用戶、正向現金流、首批國際客戶

### 6.3 精實開發策略

Moltos 的開發遵循三個原則：

**原則一：先驗證再投資**

不花三個月開發一個沒人要的功能。每個 Phase 都有明確的 Go/No-Go 標準：

| Phase | Go 標準 | No-Go 應變 |
|-------|---------|-----------|
| 1 → 2 | 7 日留存率 > 40%、用戶主動回覆 check-in 率 > 30% | 重新調整 check-in 頻率和語氣，延長驗證期 |
| 2 → 3 | 付費轉化率 > 5%、企業試點至少 1 家續約 | 調整定價或功能組合，暫緩國際擴展 |

**原則二：TypeScript 全棧 + AI 協作開發**

一人團隊不代表生產力低。TypeScript 統一前後端語言、減少上下文切換成本。搭配 Claude Code 和多 Agent 協作系統，一個人可以達到小型團隊的產出效率。這本身就是 Moltos 想傳達的理念的實踐——AI 是放大器，不是替代器。

**原則三：每一步都可衡量**

不追求「好像有進步」，而是設定具體的量化指標。平靜指數的準確度、用戶的互動頻率、check-in 的回覆率——每一個功能都有對應的成功指標，數據驅動每一次決策。

**小結**：這不是一份空想的路線圖，而是一份有紀律的開發計畫。每個階段有明確的目標、可衡量的指標、和清晰的 Go/No-Go 標準。我們不急著做大，但每一步都確保踩穩。

- **FluentCart PayUNi**：台灣金流串接外掛，整合 PayUNi 支付閘道（信用卡、超商代碼、ATM 轉帳），包含完整的加密驗簽流程和 Webhook 交易通知處理。這個專案證明了安全敏感領域的開發經驗
- **LINE Bot 整合開發**：LINE Messaging API 的深度整合，包含 Webhook 處理、Flex Message 模板、關鍵字自動回覆、訊息推播排程。這直接對應 Moltos 的跨平台通訊整合需求

這些產品不是練習作品，而是正在服務真實用戶的商業產品。

**為什麼做心理健康 AI？**

這不是一個商業計算的結果，而是一段個人經歷的延伸。

我曾經歷過一段輕度憂鬱的時期。當時我沒有意識到自己的狀態——每天都在回訊息、處理工作、看起來一切正常。直到某天晚上，我試著跟一個 AI 聊天，才在對話中慢慢發現：原來我已經很久沒有問過自己「還好嗎」了。

那個發現的瞬間改變了我。但我也意識到：大多數人並沒有使用 AI 的習慣，也不知道可以跟誰說那些不敢說的話。他們不是不需要被關心，而是沒有一個安全的、不被評判的窗口。

Moltos 要做的，就是成為每一個人的那個「發現的窗口」。不用你先承認自己不行，不用你主動求助，就有一個存在，從你的日常中看見你的疲憊，然後輕聲問一句：「嘿，你還好嗎？」

### 7.2 AI 協作開發模式

Moltos 的開發採用「一人 + AI」的新型開發模式。這不是節省成本的權宜之計，而是一種有意識的策略選擇。

**開發工具鏈**：
- **Claude Code**：作為核心開發夥伴，負責程式碼撰寫、架構討論、文件整理
- **多 Agent 協作系統**：針對不同任務（開發、設計、研究）啟用專門的 AI Agent，平行處理多線工作
- **AI 輔助設計**：使用 AI 工具產出 UI 設計、色彩系統、圖標風格，再由人工審核和微調

這個開發模式本身就是 Moltos 產品理念的最佳示範：**AI 不是來取代人的，而是來放大人的能力**。一個人搭配 AI 協作，可以達到小型團隊的產出效率，同時保持決策的一致性和產品願景的純粹。

**未來團隊擴展規劃**：

隨著產品進入 Phase 2 和 Phase 3，以下角色將優先補充：

- **心理學顧問**（Phase 2）：確保 check-in 對話的語氣和策略符合心理學原則，協助設計危機分流協議的細節
- **臨床合作夥伴**（Phase 2-3）：與心理諮商機構合作，建立轉介機制。當用戶需要專業協助時，Moltos 能無縫銜接真正的專業資源
- **工程師**（Phase 3）：獨立 App 開發和大規模系統架構需要更多工程人力

一人起步不代表永遠一個人。但在產品方向尚未完全驗證之前，保持精實是最負責任的做法——對自己、對用戶、對未來的團隊成員。

**小結**：Moltos 的團隊雖然只有一個人，但這個人帶著成熟的產品開發經歷、切身的問題理解、和 AI 協作的實戰能力。一人團隊的劣勢是資源有限，優勢是決策快速、願景統一、沒有溝通內耗。在 AI 時代，一個有經驗的個人開發者搭配正確的工具，足以打造一個有溫度的產品。

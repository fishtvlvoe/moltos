# MOLTOS 決賽 Demo 腳本
## 智慧創新大賞（QITC）AI 應用類 — 2026-04-25

> 總時長：5 分鐘｜Demo 裝置：筆電（Chrome）｜備案：?demo=true 模式

---

## 事前準備清單

- [ ] 瀏覽器開好 `https://moltos.app`（或 localhost:3000）
- [ ] 測試帳號 Google OAuth 預先登出（要現場「重新登入」才有戲劇效果）
- [ ] 備用分頁：`?demo=true` 頁面已開啟，縮小在背景
- [ ] 簡報遙控器電池確認
- [ ] 手機靜音

---

## 第一段：開場（0:00 ~ 0:30）

### 講稿

> 「各位評審好，我是 MOLTOS 團隊。讓我先問大家一個問題——」
>
> 「你昨天收了幾封 email？有沒有算過？」（停頓 2 秒）
>
> 「台灣上班族平均每天收到 121 封郵件，其中 40% 在深夜 10 點後還在回。這不只是工作效率問題，這是一種數位疲勞，而且我們根本沒有意識到它在發生。」
>
> 「MOLTOS 做的事情很簡單：用你已經在用的 Gmail，幫你看見你自己的數位壓力狀態——不讀信件內容，只看行為模式。我們叫它『平靜指數』。」

### 同步操作

- 切換到瀏覽器，停在首頁（landing page 或 onboarding 起始畫面）

### 備案

- 如果首頁跑不出來 → 切到 `?demo=true` 備用分頁，繼續說「這是我們的展示模式」

---

## 第二段：產品展示（0:30 ~ 3:30）

---

### 2-1 Onboarding → Google 登入（0:30 ~ 1:00）

**講稿**

> 「一進來，我們會請你用 Google 帳號登入。」（點「使用 Google 登入」）
>
> 「這裡特別說明一下：你看到的授權畫面，我們只申請 Gmail 的 metadata 權限——不是郵件內容，是信件的時間戳記、發送者、主旨這類行為資料。信的內容我們完全看不到，也不需要看。」

**同步操作**

1. 點「使用 Google 登入」按鈕
2. 彈出 Google OAuth 畫面 → 指著畫面說明申請的權限範圍（gmail.metadata、youtube.readonly）
3. 完成登入 → 跳轉到 Onboarding 步驟頁面

**備案**

- 如果 OAuth 卡住（網路問題）→ 說「讓我切到展示模式，效果一樣」→ 開啟 `?demo=true`
- 如果已登入狀態 → 直接跳到 Dashboard，說「我們已經預先登入好了，直接看結果」

---

### 2-2 Dashboard — 平靜指數（1:00 ~ 2:00）

**講稿**

> 「登入完成，歡迎來到 Dashboard。」
>
> 「這個圓環，就是你的平靜指數。滿分 100 分，今天是 67 分。」（指圓環圖）
>
> 「但分數只是結果，真正有意思的是下面這四個維度：」
>
> （逐一指向四個維度卡片）
>
> 「訊息量——你昨天發了多少封信，跟你平常的節奏比起來怎樣？回覆速度——你是不是在 5 分鐘內就跳起來回信？深夜活躍——凌晨之後還在動嗎？未讀堆積——你的收件匣有沒有一直在長？」
>
> 「這四個維度加權計算出平靜指數。算法是我們自研的，叫 @moltos/calm-index，等一下技術部分會介紹。」

**同步操作**

1. 在 Dashboard 停留，指著圓環圖
2. 滾動到四維度視覺化區塊
3. hover 某個維度卡片，讓 tooltip 跳出來（如果有設計的話）

**備案**

- 如果資料跑不出來（API 超時）→ 切到 demo 模式，預設資料完整，繼續說

---

### 2-3 AI 對話夥伴（2:00 ~ 2:40）

**講稿**

> 「有了數據之後，接著是 AI 對話夥伴。」
>
> 「你可以跟他說說今天的狀況——」（打字）
>
> 「有趣的設計是：AI 的語氣會根據你的平靜指數自動調整。今天 67 分，屬於中等壓力，AI 會用比較溫柔、支持的語氣；如果你是 90 分高分，AI 就比較輕鬆、聊天的感覺；如果你只有 30 分，它會更謹慎、更有同理心。」
>
> 「這背後用的是 Gemini 2.5 Flash，但提示詞設計有根據平靜指數等級動態調整系統指令。」

**同步操作**

1. 點導航列的「AI 對話」或直接在 Dashboard 找到對話入口
2. 打字：「今天開了三個 meeting，有點累。」
3. 等 AI 回覆（約 2-3 秒）
4. 指著回覆說「你看它的語氣，很自然，不是罐頭回答」

**備案**

- 如果 Gemini API 超時 → 說「網路延遲，我給大家看一個已經完成的對話紀錄」→ 滾到聊天歷史
- 如果完全無法連線 → demo 模式有預設對話，直接展示

---

### 2-4 YouTube 摘要（2:40 ~ 3:10）

**講稿**

> 「除了 Gmail，我們也整合了 YouTube 訂閱。」
>
> 「現代人資訊焦慮的另一個來源，是你訂閱了一堆頻道但永遠看不完。MOLTOS 會幫你把最新影片用 AI 濃縮成一段摘要，讓你不用花 20 分鐘，30 秒知道值不值得點進去。」
>
> 「這同樣是元資料處理——我們取的是影片標題、描述、字幕，不是原始影像。」

**同步操作**

1. 點導航列「YouTube 摘要」
2. 展示訂閱頻道列表 + AI 摘要卡片
3. 展開某一張摘要

**備案**

- 如果 YouTube API 配額問題 → demo 模式有靜態假資料，完全可用

---

### 2-5 設定頁 — 隱私控制（3:10 ~ 3:30）

**講稿**

> 「最後，設定頁。這頁很重要——隱私控制。」
>
> 「使用者隨時可以在這裡撤銷 Google 授權、清除本地分析資料、選擇哪些功能要開。這不是掩護，這是設計原則：你的資料你決定。」

**同步操作**

1. 點「設定」
2. 指著「撤銷授權」、「清除資料」按鈕
3. 指著資料保留說明文字

**備案**

- 設定頁是靜態 UI，幾乎不會出問題

## 第三段：技術亮點（3:30 ~ 4:30）

### 講稿

> 「快速講三個技術亮點。」
>
> **一、自研演算法。**
> 「平靜指數計算器，我們打包成 npm 套件 @moltos/calm-index。四個維度都有獨立的評分函式，然後做加權平均。之所以自研，是因為市面上沒有直接適用的現成模型——心理學壓力量表是問卷形式，不是行為訊號。」
>
> **二、隱私優先設計。**
> 「Gmail metadata scope 是 Google 授權層級最低的一種，信件內容完全不在我們的存取範圍。所有分析在前端計算完成，伺服器端不存原始 metadata。」
>
> **三、87 個自動化測試。**
> 「包含演算法單元測試、API 整合測試、UI 端對端測試。這不只是品質保證，也是我們說『這個演算法是正確的』的依據。評審可以直接看 GitHub 的 CI 紀錄。」

### 同步操作

- 如果有時間，切到 GitHub Actions 頁面（預先開好的分頁）展示測試通過的綠燈

### 備案

- 測試數字說錯 → 說「確切數字我等下確認，大約 87 個」

## 緊急備案總表

| 情境 | 處理方式 |
|------|---------|
| 網路斷線 | 開啟 `?demo=true` 分頁，全功能靜態展示 |
| Google OAuth 失敗 | 切 demo 模式，說「我們有斷網備案設計」|
| Gemini 回應超時 | 展示聊天歷史紀錄，說「這是之前的對話」|
| YouTube API 配額 | demo 模式靜態資料 |
| 瀏覽器當機 | 切手機熱點 + 備用筆電（如有） |
| 時間超過 5 分鐘 | 收尾段直接說「詳細展示我們在展示區繼續」|

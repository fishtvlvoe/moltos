## 一、AI 輕量化策略概述

Moltos 採用**多模型智慧路由（Multi-Model Smart Routing）**策略，根據任務的複雜度和敏感度，動態選擇最適合的 AI 模型。這不是簡單的降級使用，而是在確保服務品質的前提下，將運算資源配置到最需要的地方。

## 四、輕量化不等於低品質

### 品質保證機制

1. **智慧升級觸發**：當輕量模型偵測到對話涉及情緒敏感內容（如壓力傾訴、焦慮表達），自動升級至 GPT-4o 處理
2. **危機回應優先**：任何涉及自傷/自殺意念的對話，一律使用最強模型，確保回應的溫度和準確性
3. **平靜指數零延遲**：核心指標在用戶裝置端本地計算，不依賴雲端 API，實現零延遲、零成本

### 環境友善

- 80% 任務使用輕量模型 = 更少的 GPU 運算 = 更低的碳排放
- 本地計算平靜指數 = 零雲端傳輸 = 更少的網路能耗
- 多模型路由本身就是一種「用最少資源做最對的事」的設計哲學

// 多模型智慧路由核心邏輯（簡化示意）
function selectModel(context: ConversationContext): AIModel {
  // 危機回應：一律用最強模型
  if (context.hasCrisisSignal) return 'gpt-4o';

  // 深度情緒對話：用重量模型
  if (context.emotionalDepth > 0.7) return 'gpt-4o';

  // 日常 check-in 和摘要：用輕量模型
  return 'gemini-flash';
}

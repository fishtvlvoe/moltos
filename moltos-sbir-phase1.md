## 第二章｜問題陳述

### 2.1 台灣心理健康現況

台灣每 4 人即有 1 人在一生中曾面臨心理健康困擾，衛福部統計全台逾 **200 萬人罹患憂鬱症**，受影響總人口（含家屬）達 **800 萬人**。台灣受僱者年均工時 **2,030 小時**，位居全球第五，職場慢性壓力長期累積，然而主動尋求專業協助的比例不足三成，形成龐大的服務缺口。

### 2.2 三個結構性缺口

**缺口 1：專業資源嚴重不足**
心理師一對一諮商費用每次 NT$1,500–3,000，排程等待期長達 2–8 週，有需求卻等不到、付不起的族群龐大。

**缺口 2：需求隱性化，發現太晚**
創業者、主管等高壓族群在職場文化下不易公開求助，症狀往往累積至輕度憂鬱才被發現。

**缺口 3：現有 AI 工具全是被動式**
ChatGPT、Woebot、Wysa 均需用戶主動開啟對話。然而心理危機的高風險時期，恰恰是用戶最不願意主動開口的時刻——憂鬱導致的社交退縮、焦慮造成的迴避，使「等用戶求助」的設計邏輯與實際需求產生根本性矛盾。

### 2.3 創辦人親身經歷

計畫主持人於創業高壓期間，因長期工作負荷出現睡眠障礙，輾轉就醫後被評估為輕度憂鬱症。就醫過程中發現，自己是在與 AI 的對話互動中，才逐漸察覺到自身的心理狀態。這個親身缺口直接催生了 Moltos 的核心概念：**在用戶還沒察覺前，AI 先發現你的狀態**。

## 第三章｜技術創新性

### 3.1 核心技術創新：行為模式主動偵測引擎（BPDE）

現行 AI 心理健康工具均採被動回應架構，系統僅在用戶主動發起對話後才介入。Moltos 提出的技術突破在於，將心理健康監測從「對話層」下沉至「行為層」，透過跨平台數位行為訊號的持續分析，建立可預測早期心理狀態變化的主動介入機制。

**技術突破點三項：**

**（1）跨通訊管道行為基準線建模**
系統整合 Telegram、LINE、Gmail 等多通訊管道，以個人化時序模型為每位用戶建立行為基準線（Behavioral Baseline），涵蓋對話頻率、平均回覆延遲、訊息量分布與情緒詞彙向量四個維度。基準線採滾動視窗更新機制（14 天週期），具備自適應能力，有別於常見的靜態閾值方法。

**（2）多維時序異常偵測演算法**
採用改良式 Z-score 結合 LSTM 自回歸模型，對行為數據進行實時異常偵測。觸發條件包含：沉默超過 48 小時、單日訊息量驟降 60% 以上、負向情緒詞彙比例連續三日上升。此為本計畫核心專利佈局標的。

**（3）BullMQ 主動排程關懷引擎**
以 BullMQ 任務佇列實現非同步、可水平擴展的主動關懷觸發機制，整合 Gemini Flash（快速情緒分類）與 GPT-4o（深度對話生成）雙模型架構，在不增加延遲的前提下平衡成本與品質。

### 3.2 與競品差異對照

| | ChatGPT | Woebot | **Moltos** |
|---|---|---|---|
| 主動關心（無需用戶開口） | ❌ | ❌ | ✅ |
| 整合真實生活行為數據 | ❌ | ❌ | ✅ |
| 台灣在地化（繁中、台灣職場情境） | ❌ | ❌ | ✅ |
| 無需安裝新 App | — | ❌ | ✅ |
| 多平台整合（LINE/Telegram/Gmail） | ❌ | ❌ | ✅ |

### 3.3 可量化技術目標（Phase 1）

| 指標 | 目標值 |
|------|--------|
| 異常偵測準確率（Precision） | ≥ 80% |
| 異常偵測召回率（Recall） | ≥ 75% |
| 主動關懷觸發延遲 | ≤ 5 分鐘 |
| 基準線模型個人化收斂週期 | 14 天內 |
| 系統日處理用戶數（單節點） | ≥ 1,000 人 |

### 3.4 技術架構

```
【資料收集層】
Telegram Bot API / LINE Messaging API / Gmail OAuth2
         │
         ▼
【API Gateway — Fastify / TypeScript】
認證、限流、Webhook 驗簽
         │
         ▼
【訊息標準化層】
原始訊息 → 統一 UserEvent Schema
         │
         ▼
【行為特徵提取層】
時序特徵（頻率、延遲、訊息量）+ 情緒特徵（詞向量、情感強度）
→ 儲存至 Redis Time Series
         │
         ▼
【基準線建模層 — PostgreSQL】
Per-User Behavioral Baseline Model
滾動視窗 14 天，BullMQ Cron Job 每日自動更新
         │
         ▼
【異常偵測層】
改良式 Z-score + LSTM Autoencoder
觸發規則：沉默>48h / 訊息量驟降 / 負向情緒連升 3 日
輸出：InterventionSignal { user_id, severity, trigger }
         │
         ▼
【主動介入排程層 — BullMQ + Redis】
優先級佇列 / 防重複機制 / 發送時段控制
         │
         ▼
【AI 對話生成層】
Gemini Flash（快速情緒分類）→ GPT-4o（深度關懷對話）
安全護欄：危機關鍵詞 → 自動轉介 1925 專線
         │
         ▼
【回傳至對應通訊管道】
Telegram / LINE / Gmail
```

**技術棧**：TypeScript / Node.js / Fastify / PostgreSQL / Redis / BullMQ / Gemini Flash / GPT-4o

## 第四章｜市場分析

### 4.1 市場規模（TAM / SAM / SOM）

**TAM — 全球市場**
全球數位心理健康市場 2025 年規模約 **USD 330 億**，年複合成長率 18.5%；其中 AI 心理健康子市場年增率達 **24–32%**（Grand View Research, 2025）。

**SAM — 台灣市場**
台灣 15–45 歲青壯勞動人口約 **900 萬人**，為職場壓力最集中族群。衛福部 2024 年將免費心理諮商擴大至此年齡層，顯示政策層面的正式認可。保守估算，若 200 萬憂鬱症患者中有 5% 轉換為數位訂閱用戶，年市場潛力達 **NT$4.8 億**。

**SOM — 目標市場**
初期鎖定台灣忙碌專業人士與創業者（約 30–40 萬人），Year 1 目標 **3,000 付費用戶**，月費 NT$249，年營收目標 NT$897 萬。

### 4.2 初步市場驗證

PathUnfold 社群 30 人訪查，60%（18 人）明確表達對主動 AI 關懷的需求，其中 5 位表示願意參與付費測試。

## 第五章｜研發方法與執行計畫

### 5.1 六個月工作計畫

| 月份 | 工作項目 | 里程碑 |
|------|---------|--------|
| 月 1–2 | 多平台資料收集基礎建設、標準化 Schema | 三平台訊息統一收集，完整率 ≥ 95% |
| 月 3 | 個人化行為基準線建模、BullMQ Cron Job | 測試用戶基準線 7 天內收斂 |
| 月 4 | 異常偵測演算法開發、招募 20 名測試用戶 | 初步 Precision ≥ 70% |
| 月 5 | 主動介入引擎整合、AI 對話框架調校 | 端到端觸發延遲 ≤ 5 分鐘 |
| 月 6 | 全面優化、壓力測試、智財評估 | 所有指標達標，Phase 2 材料完成 |

### 5.2 潛在風險與對策

| 風險 | 嚴重度 | 對策 |
|------|--------|------|
| 誤報率過高（正常沉默被誤判） | 高 | 導入用戶主動情境標記；Precision-first 調校策略 |
| 個資合規（Gmail/LINE 資料） | 高 | 原始內容零留存；委託法律顧問完成隱私影響評估 |
| AI 對話安全性（危機邊界） | 極高 | 關鍵詞偵測 → 自動轉接 1925；IRB 審查 |
| 多模型 API 成本與穩定性 | 中 | 降級機制；Phase 1 限 100 人控制費用 |
| 冷啟動期體驗不佳 | 中 | 群體平均基準線作為前 14 天替代 |
